About the Project

Addressing a Real Educational Challenge

The Problem

Many secondary schools in Uganda lack fully equipped science laboratories. Students often learn practical concepts theoretically without hands-on experience, limiting their understanding and exam preparedness.

Why We Built It

As a final-year team, we wanted to bridge this gap using affordable, immersive technology making quality science practicals accessible to students regardless of their school's resources.

Our Solution

The Virtual Reality Science Practical System (VRSPS) combines a Meta Quest VR application with a web management portal delivering interactive lab simulations and tracking student progress.

Expected Impact

We aim to improve science practical comprehension, increase student engagement, and provide teachers with tools to monitor and assess learning outcomes in resource-limited settings.

Project Overview

What We Built

Project Goal

Deliver an affordable, immersive VR-based science practical system that supplements traditional laboratory education in underserved schools.

Development Journey

From Idea to Demonstration

Idea Generation

Identified the lack of science lab resources in Ugandan schools and explored VR as a scalable, cost-effective alternative for practical learning.

Requirements Gathering

Interviewed teachers and students, reviewed curriculum needs, and defined functional requirements for both the VR app and web portal.

System Design

Created architecture diagrams, database schemas, UI wireframes, and VR interaction flows for the complete VRSPS ecosystem.

Development

Built the Unity VR experiments, Next.js web portal, MySQL database, and API integrations in parallel sprints across the academic year.

Testing

Performed unit, integration, and user acceptance testing with sample student accounts and real VR hardware sessions.

Deployment

Deployed the web portal and configured the database, environment variables, and Unity API connectivity for demonstration readiness.

Final Demonstration

Presented the complete system to supervisors and evaluators, demonstrating end-to-end student lab workflows in VR and on the web portal.

Challenges and Solutions

What We Learned Along the Way

Challenge

Limited access to VR hardware during development

Not all team members had continuous access to Meta Quest headsets, slowing VR testing cycles.

Solution

Established a shared testing schedule, used Unity Editor simulation mode for early development, and documented VR issues in a shared tracker.

Lessons Learned

Early planning for hardware access and modular VR design reduces blockers in immersive projects.

Challenge

Integrating Unity VR with the web portal

Session data, authentication, and experiment progress needed reliable communication between two separate platforms.

Solution

Designed a secure REST API with API-key authentication and JWT tokens for student VR sessions, with Prisma-managed persistence on the web side.

Lessons Learned

Clear API contracts and thorough documentation are essential when bridging game engines and web backends.

Challenge

Designing for low-resource school environments

The system needed to work reliably with limited internet connectivity and shared devices.

Solution

Optimized the portal for performance, used offline-capable VR sessions where possible, and designed simple, intuitive interfaces for non-technical users.

Lessons Learned

User-centered design must account for real-world infrastructure constraints, not just technical ideals.
